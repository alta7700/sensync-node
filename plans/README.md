# plans

Папка хранит прикладные исследования и планы по отдельным направлениям развития `sensync2`.

Как работает:
- каждый файл в папке описывает одну задачу или один интеграционный трек;
- в этой же папке живут теоретические планы вычисляемых обработчиков сигналов, если решение ещё рано фиксировать в коде;
- если задача упирается в внешний контракт, библиотеку или формат данных, в плане должны быть ссылки на источники;
- если по результатам работы меняется решение или стек, соответствующий план нужно актуализировать в том же изменении.

Как взаимодействует с остальными модулями:
- планы опираются на контракты из `packages/core`;
- планы по устройствам и форматам данных описывают будущие плагины для `apps/runtime` и `packages/plugin-sdk`;
- при превращении плана в код нужно обновлять `README.md` затронутых модулей и, если требуется, `CONTRACTS.md` / `SCHEMA.md`.

Текущие рабочие планы:
- `adapter-kit.md`
- `plugin-kit.md`
- `plugin-multi-instance.md`
- `processor-kit.md`
- `test-coverage.md`
- `timeline-reset.md`
