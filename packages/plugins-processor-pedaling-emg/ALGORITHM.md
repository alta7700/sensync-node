# ALGORITHM

Документ фиксирует выбранную `v1` схему processor-а `pedaling-emg`.

## Граница ответственности

- `TypeScript` отвечает за:
  - выбор рабочей `gyro`-оси;
  - поиск цикла по zero-crossing;
  - фазовую нормализацию;
  - вырезание короткого `EMG`-сегмента;
  - lifecycle/runtime integration.
- `Python` отвечает за:
  - плотную обработку короткого `EMG`-сегмента;
  - baseline / thresholds;
  - onset / offset;
  - confidence и status результата.

## Фазовый трекер `v1`

- Цикл определяется по positive zero-crossing одной выбранной `gyro`-оси.
- Ось выбирается по устойчивости периодики, а не по амплитуде.
- Для `v1` применяем простую practical-схему:
  - удаление DC через медленный EMA baseline;
  - сглаживание через быстрый EMA;
  - оценка устойчивости по интервальным характеристикам zero-crossing.
- Переходы грубой фазы публикуются как `label-batch`, а не как full-rate binary signal.

## `EMG`-анализ `v1`

- В `Python` уходит только короткий сегмент вокруг ожидаемого окна активности.
- Базовый пайплайн:
  - band-pass;
  - rectification;
  - envelope / RMS;
  - adaptive baseline;
  - hysteresis threshold;
  - minimum-state-duration.
- Производная огибающей используется только как дополнительный признак, а не как единственный критерий onset/offset.
- В Python отправляется только окно с полным временным покрытием; если буфер `EMG` не содержит начало или конец сегмента, `TS` не формирует request вовсе.

## Деградация `v1`

- Если Python worker не поднялся или упал на restart, `TS`-фазовый слой не падает вместе с ним:
  - `phase` и `phaseConfidence` продолжают публиковаться;
  - `activity` принудительно возвращается в `0`;
  - `emgConfidence` публикуется как `0`.
- Если периодика по `gyro` пропала и цикл больше не замыкается, `phaseConfidence` всё равно материализуется наружу и падает к `0`, чтобы UI и запись не оставались с устаревшим высоким значением.

## Практические ограничения `v1`

- Окно активности задаётся профилем и не обучается по истории автоматически.
- Абсолютная фазовая интерпретация зависит от фиксированного mounting и выбора zero-crossing якоря.
- `v1` предназначен для стабильного live-сценария с фиксированным расположением датчика, а не для произвольного переиспользования без калибровки.
